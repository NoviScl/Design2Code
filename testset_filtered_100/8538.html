1. Topics in Topic Modeling

What was topic modeling again?

  • A method of discovering the latent semantic structure of large collections of texts
  • Unsupervised, language-independent, based on distributional semantics
  • A topic model describes a set of topics with their words and their distribution in the collection
  • Useful e.g. for resource discovery, recommender systems and thematic analysis

Words, documents, topics

(David Blei, "Introduction to Probabilistic Topic Modeling", 2012)

What are topics? (Blei)


"We formally define a topic to be a distribution over a fixed vocabulary."

"Probabilistic topic models are a suite of algorithms whose aim is to discover the hidden thematic structure in large archives of documents."

"A topic model [...] discovers a set of “topics” — recurring themes that are discussed in the collection — and the degree to which each document exhibits those topics."


David Blei 2012 (Computer Scientist)

My take on topics

  • It is true that, depending on preprocessing, 'topics' do not always correspond to clear, abstract themes
  • However, in most cases, topics do show some sort of semantic relation between the words comprising them
  • However, this semantic relation is not limited to a shared abstract theme
  • Rather, there are several types of topics, some thematic, some non-thematic

Types of topics: requirements

  • Types should not be defined based on meaning (not, e.g.: nature vs. culture topics)
  • Types should not be defined based on word categories (not, e.g. noun vs. verb topics)
  • A useful basis could be the narrative function of the topics: narrative, descriptive, argumentative; +discursive.

Types of topics

  • Descriptive: setting, physical, personnel
  • Narrative: motives, ?affects
  • Argumentative: abstract, ?meta
  • Discursive: language, ?rhetorics

Type: descriptive_personnel

  • Words describing characters in a functional or relational way (not just character names)
  • E.g. family members, social rank terms, professional types, etc.

Example: descriptive_personnel (fr)

bishop, sire, serf

Example: descriptive_personnel (en)

lord, earl, lady

Type: physical description

  • Words used to describe the body and/or clothes of a person

Example: physical description (fr)

leg, foot, body

Example: physical description (en)

little, hand, hair, white, finger

Type: descriptive_setting

  • Words describing fictional spaces in which events take place
  • E.g. interiors, exteriors, landscapes

Example: descriptive_setting (fr)

bed, room, table, window, room

Example: descriptive_setting (en)

wood, tree, ground, leave

Type: narrative_motive

  • Words used to describe some typical sequence of events involving characters in a given setting
  • E.g.: taking the train, reading/writing letters, walking in town

Example: narrative_motive (fr)

doctor, doctor, sickness

Example: narrative_motive (en)

dinner, table, bread, eat, food

Type: argumentative_abstract

  • Words used to discuss some philosophical, political or social theme

Example: abstract theme (fr)

memory, past, dream

Example: abstract theme (en)

life, live, death, die, world ('existence')

Type: discourse_language

  • Words belonging to a certain register, sociolect, dialect or another language

Example: discourse_language (fr)

kid, s#%t, butt

Example: discourse_language (en)

thine, thou, art

Finally: other_other

  • Topics that don't fit any of these categories

Example: other_other (fr)

face, corner, fist, old, shoulder

Example: other_other (en)

dog, seem, realize, window

Example: other_other (en)

morning, send, day, return

3. Some reasons and consequences

Some reasons

  • Literary texts are non-expository (they talk about more than concepts and theories)
  • Implicit / symbolic instead of explicit (topic modeling sees everything as explicit, misses implicit things)
  • Literary texts have background and foreground information (topic modeling doesn't distinguish these)

Some consequences

  • Topics != themes
  • Topic modeling does work in literary studies
  • Discovers more than themes
  • Literary studies are well-equipped for this

4. First analyses
(proof of concept)

Basic hypothesis

  • If the typology is theoreticall well-founded,
  • and, if the typology is perceptually helpful,
  • then, it should be possible to find formal characteristics distinctive of each type
  • and if they exist, automatic classification should work with high accuracy

Potentially useful features (implemented)

  • Overall probability in the model (score)
    Mallet output - average: 0.039
  • Topic coherence scores (ca, cv, umass scores)
    using Palmetto - average: 0.20/0.44/-3.87
  • Amount of nouns, verbs and adjectives (proportion)
    TreeTagger - average: 0.50/0.32/0.18
  • Skewness, kurtosis, variance, stdev of word probabilities
    Python scipy

Other features
(not implemented)

  • WordNet categories and hierarchy level of words
  • Sentiment scores: mean and variance
  • Probably not useful: topic words themselves (may work but unlikely to generalize)

6. Conclusion

What have we found out?

  • Literary topics are (probably) special
  • Topics come in different types
  • It remains an open question whether different types of topics have formal, statistical characteristics

Next steps

  • Build a more systematic, theoretically-founded typology
  • Use larger models and crowdsource the classification of topics
  • Generate more features: WordNet categories; distributional characteristics; etc.
  • Improve the classification methods: feature scaling, other classifiers, etc.
